====test====
Given form data expected in request body then return 422 when that body is invalid.
====spec====
openapi: '3.0.1'
paths:
  /path:
    post:
      responses:
        200:
          schema:
            type: string
      requestBody:
        content:
          application/x-www-form-urlencoded:
            schema:
              type: object
              properties:
                id:
                  type: integer
                status:
                  type: string
                  enum:
                    - open
                    - close
              required:
                - id
                - status
====server====
mock -p 4010
====command====
curl -i -X POST http://localhost:4010/path -H "application/x-www-form-urlencode" --data-urlencode "status=ooopsie!"
====expect====
HTTP/1.1 422 Unprocessable Entity
content-type: application/problem+json; charset=utf-8
content-length: 0
Connection: keep-alive

{"type":"https://stoplight.io/prism/errors#UNPROCESSABLE_ENTITY","title":"Invalid request body payload","status":422,"detail":"Your request body is not valid and no HTTP validation response was found in the spec, so Prism is generating this error for you.","validation":[{ code: 'required', message: "should have required property 'id'", severity: 'Error' }, { code: 'enum', location: ['body', 'status'], message: 'should be equal to one of the allowed values', severity: 'Error' }]}
